# Codebase Structure

**Analysis Date:** 2026-03-23

## Directory Layout

```
videoconverter/
├── src/                  # Main application source code
│   ├── assets/           # UI assets, icons, etc.
│   ├── core/             # Business logic and ffmpeg wrappers
│   ├── ui/               # PyQt6 windows, workers, and themes
│   ├── utils/            # Configurations, logging, and helpers
│   └── main.py           # Application entry point
├── tests/                # Pytest suites
│   ├── unit/             # Unit tests
│   └── (integration)     # Integration and specific case tests
├── .planning/            # Planning documents
│   └── codebase/         # Generated codebase documentation
├── pyproject.toml        # Application configurations and lint tools
├── requirements.txt      # Python package dependencies
├── requirements-dev.txt  # Dev requirements (pytest, testing tools)
├── start.bat             # Windows launcher
├── check_ffmpeg.bat      # CLI script for ffmpeg validations
└── config.json           # Application config store (generated at runtime)
```

## Directory Purposes

**src/core:**
- Purpose: Application's functional backend. Houses the video processing engine.
- Contains: Conversion logic, codec enumerations, and runtime exceptions.
- Key files: `converter.py`, `exceptions.py`.

**src/ui:**
- Purpose: The graphical frontend using PyQt6.
- Contains: Windows definitions, theme overrides, and multithreading worker implementations.
- Key files: `main_window_qt.py`, `worker.py`, `preview_window_qt.py`.

**src/utils:**
- Purpose: Utilities used across core and UI modules.
- Contains: Config managers, helpers (like ffprobe callers), and simple caching logic.
- Key files: `config.py`, `helpers.py`, `logger.py`.

**tests:**
- Purpose: Quality assurance.
- Contains: Unit tests, structural integration tests, and test fixtures (`conftest.py`).

## Key File Locations

**Entry Points:**
- `src/main.py`: Bootstraps the PyQt6 application loop.
- `start.bat` / `start.sh`: Convenience wrapper scripts.

**Configuration:**
- `pyproject.toml`: Configures `ruff`, `mypy`, and `pytest` paths and markers.
- `requirements.txt`: Python package dependency list.

## Naming Conventions

**Files:**
- Standard Python `snake_case` across all modules: `main_window_qt.py`, `thumb_cache.py`.
- Test files prefixed with `test_`: `test_auto_delete.py`, `test_ui.py`.

**Identifiers:**
- Classes: `PascalCase` (e.g., `MainWindow`, `ConversionWorker`, `Converter`).
- Methods/Functions: `snake_case` (e.g., `calculate_bitrates()`, `convert_single_file()`).
- Constants: `UPPER_SNAKE_CASE` (e.g., `RESOLUTION_1080P`, `DEFAULT_CRF_VALUE`).

## Where to Add New Code

**New UI Features:**
- Windows/Dialogs: Add to `src/ui/`, extending `QMainWindow` or `QDialog`.
- Threads/Async jobs: Add a new `QRunnable` inside `src/ui/worker.py`.

**New Video Processing Features:**
- Logic: Add to `src/core/converter.py` logic conditions.
- Extractions/Probes: Add helper functions to `src/utils/helpers.py`.

## Special Directories

**venv:**
- Virtual environment for Python (excluded from source control).

**.pytest_cache / __pycache__:**
- Cache directories generated by Python and Pytest executions.

**.planning/codebase:**
- Generated documentation artifacts detailing the structure and concepts of the application.
